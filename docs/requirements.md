# 入库操作 — 需求规格说明书

> 基于前端演示（`/#/ingest`）的完整功能描述。

---

## 整体流程

入库操作分为 **4 个步骤**，用户依次完成：

```
档案包的获取 → 检测 → 著录 → 确认入库
```

---

## 1. 档案包的获取

### 1.1 系统打通模式（推荐）

如果接收审核系统与档案馆藏系统之间可以数据互通：

- 接收系统审核端点击 **【入库】** 按钮
- 档案馆藏系统接收推送的档案信息
- 在入库界面的"上传"步骤中，**直接查看到**该条项目信息（项目名称、单体数、案卷数等）
- 无需手动上传档案包

### 1.2 系统未打通模式（手动导入）

如果两套系统之间不能打通：

- 在馆藏系统的入库界面，通过 **【上传档案包】** 按钮手动选取本地的档案包文件
- 系统解析档案包后，在上传记录列表中展示项目信息
- 支持批量上传、重新上传

---

## 2. 检测

检测模块保持现有逻辑，无需修改。

- 选择一条上传记录，点击 **【检测】** 进入检测页面
- 检测档案包的真实性、完整性、可用性、安全性
- 检测通过后可进入著录步骤

---

## 3. 著录

著录是整个系统的核心功能模块。

### 3.1 四级元数据编辑

支持对 **项目级/工程级/案卷级/文件级** 四级元数据（著录单）进行修改：

| 层级 | 著录单内容 | 标题配色 |
|------|-----------|---------|
| **项目级** | 基本信息、专业记载（含财务汇总）、档案状况、保管信息、排检与编号 | 🟦 `bg-blue-200` |
| **工程级** | 基本信息、专业记载、档案状况、保管信息、排检与编号、其他 | 🟩 `bg-emerald-100` |
| **案卷级** | 专业记载、档号、工程档案整理记录、档案状况、备注 | 🟧 `bg-amber-50` |
| **文件级** | 文件基本信息、存放处、保管信息、物理形态、其他 | ⬜ `bg-slate-500` |

**排版样式**：单张带边框表格，字段名浅蓝底色（`#f0f9ff`），值区白底。各层级标题颜色区分。

**档号规置**：4 个层级的"档号"及相关字段（总登记号、大类流水号、案卷档号）在著录阶段**不可编辑**（灰色底纹），在第 4 步确认入库时批量生成。

### 3.2 文件级预览

- 选中文件级节点后，右侧顶部显示 **【著录】/【预览】** 两个标签页
- 点击 **【预览】** 可查看档案文件内容
- 支持的格式：PDF（通过浏览器嵌入预览）、OFD、JPG 等
- 演示阶段可通过本地上传的 Data URL 或服务端 URL 预览

### 3.3 案卷级添加文件

- 在任一文件级节点的操作栏点击 **【添加文件】** 按钮
- 弹出新增文件表单，包含完整文件级元数据
- 必填项：选择本地 PDF 文件、填写文件题名
- 保存后：数据持久化到 localStorage，左侧目录树**实时刷新**，新文件自动选中

### 3.4 文件级管理操作

每个文件级节点提供以下操作（位于右侧著录单顶栏）：

| 操作 | 功能 |
|------|------|
| **上移** | 与同一案卷下的上一个文件交换顺序 |
| **下移** | 与同一案卷下的下一个文件交换顺序 |
| **更换文件** | 重新选择本地文件替换当前文件的 PDF/OFD |
| **删除文件** | 确认后从树中移除该文件节点 |

操作后左侧树**实时刷新**，排序通过 `localStorage` 持久化。

### 3.5 已入库档案的著录修改

已确认入库的档案，仍然可以重新进入著录步骤修改其元数据信息（后续开放功能）。

---

## 4. 确认入库

### 4.1 档案类型选择

入库前需要确认档案的一类/二类/三类：

| 层级 | 名称 | 说明 |
|------|------|------|
| **一类** | 建筑工程 | 项目一级分类，从系统预置类型树中选择 |
| **二类** | 民用建筑/工业建筑等 | 二级分类 |
| **三类** | 住宅/厂房/仓库等 | 三级分类 |

### 4.2 档号生成规则

确认入库时，系统根据档号规则批量生成：

```
I1-1-{项目流水号}-{工程order_no}-{案卷order_no}-{文件order_no}
```

- **项目级档号**：`I1-1-029605`
- **工程级档号**：`I1-1-029605-{3位工程序号}`
- **案卷级档号**：`I1-1-029605-{工程序号}-{案卷序号}`
- **文件级档号**：`I1-1-029605-{工程序号}-{案卷序号}-{3位文件流水号}`

案卷级自动生成：总登记号（6位递增）、大类流水号、案卷档号。

### 4.3 确认与取消

| 操作 | 功能 |
|------|------|
| **取消入库** | 返回著录步骤，可继续编辑 |
| **确认入库** | 生成档号、完成入库，更新任务状态为"入库成功" |
