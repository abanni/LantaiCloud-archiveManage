# 综合查询需求文档

## 1. 功能目标

用户输入一个或多个关键词，系统在项目/工程/案卷/文件4级元数据中联合检索，结果按档案树形结构展示。

## 2. 搜索交互

| 元素 | 说明 |
|------|------|
| 搜索框 | 单行输入，多个关键词用空格分隔 |
| 搜索按钮 | 点击或回车触发搜索 |
| 重置按钮 | 清空关键词和结果，回到初始状态 |

输入过程不触发搜索，必须点击搜索按钮或按回车。

## 3. 搜索逻辑

### 3.1 关键词匹配

- 每个关键词在当前层级的所有可搜索字段中模糊匹配（`includes`）
- 匹配不区分大小写
- 日期字段、数字字段不参与搜索

### 3.2 跨层级过滤（核心逻辑）

搜索不是"所有关键词命中同一节点"，而是**逐层缩小范围**：

```
关键词A → 命中项目P（建设单位含A）
关键词B → 命中案卷V（案卷题名含B，且V在P的子树中）
→ 结果：P → ... → V
```

实现方式（3阶段）：
1. **直接匹配**：遍历所有节点，记录每个节点命中了哪些关键词
2. **子树聚合**：自底向上，计算每个节点的子树中包含哪些关键词
3. **过滤保留**：只保留子树覆盖全部关键词的分支

### 3.3 搜索范围

每个层级只搜索有意义的文本字段：

| 层级 | 可搜索字段 |
|------|-----------|
| 项目级 | 项目名称、项目简称、建设/施工/监理/勘察/设计单位、立项批准文号、各类许可证号、项目地点、移交单位、负责人、经办人、附注 |
| 工程级 | 单位工程名称、文件名称、施工单位、质量安全监督单位、规划/施工许可证号、移交单位、附注 |
| 案卷级 | 案卷题名、文件名称、案卷类型、编制单位、载体类型、主题词、整理人、审核人、备注、附注 |
| 文件级 | 文件题名、文件名称、文(图)号、责任者、缩微号、摘要、主题词、备注、附注 |

### 3.4 字段优先级

标题类字段（项目名称、案卷题名、文件题名）优先级高于其他字段，排序时靠前。

## 4. 结果展示

### 4.1 树形结构

- 结果按档案层级树形展示：项目 → 工程 → 案卷 → 文件
- 只展示子树覆盖全部关键词的分支，无命中分支不显示
- 所有结果节点默认展开

### 4.2 节点标识

| 标识 | 说明 |
|------|------|
| 层级徽标 | 项目级(蓝)、工程级(绿)、案卷级(橙)、文件级(灰) |
| 关键词高亮 | 节点名称中命中的关键词用黄色背景标记 |
| 命中字段 | 直接命中的节点显示 `← 字段名`（如 `← 编制单位`） |
| 直接命中背景 | 直接命中的节点有浅黄色背景，与仅因子孙命中而显示的节点区分 |

### 4.3 排序

- 直接命中关键词数多的节点排在前面
- 同级节点按字段优先级排序（标题 > 单位名 > 附注）

### 4.4 自动预览

搜索结果出来后，自动选中第一个直接命中的节点，右侧预览面板显示其元数据。

## 5. 预览面板

| Tab | 说明 |
|-----|------|
| 元数据 | 展示选中节点的全部字段（只读） |
| 预览 | 仅文件级可用，展示PDF预览 |

面板宽度可拖拽调整（300px~800px）。

## 6. 利用清单

- 文件级和案卷级节点可加入利用清单
- 利用清单以浮动徽标显示数量
- 点击可跳转到利用模块

## 7. 空状态

| 场景 | 显示 |
|------|------|
| 未搜索 | "输入关键词点击搜索" |
| 搜索无结果 | "未找到同时包含所有关键词的档案" + "尝试减少关键词或使用更短的词" |
